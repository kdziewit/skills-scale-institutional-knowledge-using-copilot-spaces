# OctoAcme Project Management Docs

This folder contains the process documentation for how OctoAcme plans, delivers, and improves cross-functional projects. These guides are intended to help new teammates quickly get up to speed and serve as a reference throughout the project lifecycle.

## Brief Summary of OctoAcme Project Management Processes

- **Principles:** OctoAcme follows five core principles — customer-first, iterative delivery, clear ownership, data-informed decisions, and psychological safety.
- **Team rhythm:** Teams run daily standups, weekly delivery syncs, and monthly stakeholder updates. Sprint/milestone demos are held at the end of each iteration.
- **Workflows:** Work moves through a project board with columns: Backlog → Ready → In Progress → In Review → QA → Done. Pull requests are kept small, linked to issues and acceptance criteria, and require CI to pass and at least one approval before merging.
- **Key artifacts:** Project Charter / One-pager, Roadmap and Release Plan, Sprint/Iteration Backlog, Acceptance Criteria & Definition of Done, Risk Register, and Retrospective notes.
- **Quality and testing:** Each increment includes unit, integration, and smoke tests, as well as security scanning, before being considered done.
- **Release practices:** Releases follow defined pre-release requirements, a deployment checklist, and a documented rollback plan. Release notes are published for each release.
- **Continuous improvement:** After each release or milestone, the team holds a retrospective to capture learnings and track action items.
